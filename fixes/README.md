# fixes/

Documented solutions to non-obvious bugs and tricky problems encountered in
ViSiON/3 development.

Before deep-diving on a bug, check here for a similar past issue. When you
resolve a complex problem, add a descriptive `.md` file (e.g.
`YYYY-MM-DD-short-slug.md`) detailing:

1. **Problem** — the observed symptom and where it surfaced.
2. **Investigation** — how the root cause was found.
3. **Solution** — the fix, and why it works.

This directory is referenced from `CLAUDE.md` (Debugging & Troubleshooting).
